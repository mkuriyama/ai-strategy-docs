## 1. このプロジェクトの目的

本プロジェクトは、生成AI を業務に実践的に取り入れ、ROI の高いユースケースを
見極めて展開することを目的とします。まずは小さな PoC から始め、KPI で効果を
測定しながら段階的に拡大します。

## 2. 基本となる考え方

- **小さく始める**: いきなり大規模導入せず、PoC で実現可能性を確認します。
- **効果を測る**: KPI を事前に定義し、ROI を継続的に評価します。
- **安全に使う**: ガードレール を設け、ハルシネーション のリスクに備えます。

## 3. はじめての一歩

1. **用語に慣れる**: まず[用語集](glossary.md)に目を通し、LLM や 生成AI、
   プロンプト といった基本語を押さえます。
2. **試してみる**: 実際に プロンプト を入力し、出力を観察します。出力が
   安定しない場合は プロンプトエンジニアリング の工夫を加えます。
3. **自社データを活かす**: 社内文書を活用したい場合は、RAG や 埋め込み、
   ベクトルデータベース の仕組みを検討します。

## 4. つまずきやすいポイント

!!! warning "ハルシネーションに注意"
    LLM は事実と異なる内容を生成すること（ハルシネーション）があります。
    重要な情報は必ず一次情報で裏取りし、グラウンディング を意識しましょう。

!!! tip "トークンとコストの関係"
    料金や入力上限は トークン 数に依存します。長文を扱う際は
    コンテキストウィンドウ の上限に注意してください。

## 5. もう一歩進める

業務システムへ組み込む場合は API 連携を検討します。複数の手順を自律的に
実行させたい場合は エージェント の活用も選択肢になります。画像や音声も扱う
なら マルチモーダル 対応モデルを検討しましょう。既存モデルを特定業務へ
最適化したい場合は ファインチューニング を比較検討します。
